Introduction

Arduino Uno is a very interesting board to getting started with electronics, robotics and 3D printing projects, among others... It's a low cost board, very easy to use, with applications to many different contexts. It encourages, from kids and adults, to understand fundamentals of electronics and programming. Due to it's wide set of instructions and inputs and outputs, most of the basic projects can be overcome with Arduino Uno.

Hardware description
Power, operating voltage and Reset
Arduino Uno can be powered, through the power connector, with a voltage between 7V and 12V (recommended). Internally, the board adapts this voltage into a stable power signal of 5V, using the voltage regulator.

The board can be also powered through the USB connector, which drive also 5V to the microcontroller and power pins. 

Thus, the operating voltage, is 5V, which implies that all digital inputs and output pins, as well as analog input pins will deal with signals between 0V and 5V. The 0V signal can be easily identified in the board with the label GND, from ground.

Arduino Uno board provides a power pin of 3.3V to connect devices that might require this voltage, but this voltage comes from an internal voltage regulator of the microcontroller with limited current, so use with caution.

Arduino Uno's board has a reset button that will allow us to restart the program from the beginning.

Microcontroller Unit and Clock
The brain of Arduino Uno's board is the Microcontroller Unit (MCU). It is based on the well-known ATmega328P processor which has 14 digital pins and 6 analog input pins. A microcontroller covers up the equivalent function of a CPU in our computer. The main difference is that a MCU does not have an operating system. Another difference is that a MCU is usually very limited in terms of computing processing compare to a CPU. However, the main advantage is that access to low-level signals is straightforward.

Each digital pin can drive a maximum current of 20mA, which means that if we intend to connect a device that requires more current, for instance a DC motor, we will need a driver to actually provide the current, typically using transistors or relays.

Microcontroller's clock is used for timing your program instructions. On every clock tick or cycle, the microcontroller executes a new instruction. The clock's frequency is 16 Mhz, which means that it produces 16.000.000 ticks every second. This gives you the capability of executing 16.000.000 instructions per second, wow!!! However, this is not entirely true... Many instructions require more than one clock cycle to complete, but even though we can still execute them very fast, as if they were instantaneously executed.

Pins
Arduino Uno has 14 digital pins (named as D0-D13) and 6 analog input pins (named as A0-A5). Indeed, analog pins A0-A3 can be used as digital pins too. Digital pins can be configured either as input or output. Digital inputs can measure a voltage at their pin which can be 0V or 5V, these are the two possible states of a digital input: LOW or HIGH. Similarly, digital outputs can be set to two possible states LOW state and HIGH state, which in the end produce 0V or 5V voltage at their pin. On the contrary, analog inputs can measure any voltage between 0V and 5V, which in turns is transformed into an integer number between 0 and 1023 (using the analog-digital converter, a.k.a. ADC).

We can use digital and analog pins to connect a wide variety of sensors and actuators. Depending on the provided/required signal type, they will have to be connected to a digital pin or an analog pin. This is usually especified on the description of each sensor or actuator. Typically, we can connect sensor such as buttons, switches, potentiometers, presence detectors, infrared, microphones, temperature sensors, etc.) or actuators such as relays, LEDs, LCDs, DC motors, stepper motors, speakers or buzzers, transistors, etc.

Pins D0 and D1 are also known as RX and TX, respectively. They are involved in the USB serial communication to transmit/receive data to/from our computer. It is not convinient to use those pins in your project unless you know what you are doing. The Serial library of Arduino IDE uses those pins for transmiting and receiving data.

Pins D3, D5, D6, D9 D10 and D11 can be used as conventional digital input or digital output pins or they can be used as analog outpus or PWM, a periodic signal typically used for controlling actuatos such as DC motors among others.

Pins D2 and D3 can be used as conventional digital input or digital output pins or they can be used as interrupt input pins, a signal that triggers an interrupt on Arduino Uno's MCU to attend to a prioretized task.

Pins A0-A6 are used as analog input pins, although pins A0-A3 can be used as digital pins too (either digital input or digital output). These pins can not be used as analog outputs!
